Install

Cookerを自身でコンパイルする場合は、ローカルにgit,node,npm,ruby,gem,jekyll,grunt,bowerがインストールされている必要があります。

{% highlight text %} $ git version $ node -v $ npm -v $ ruby -v $ gem -v $ jekyll -v $ grunt -version $ bower -v {% endhighlight %}

Step 1

作業用ディレクトリで$ git clone。

{% highlight text %} $ cd working-dir $ git clone https://github.com/teq-inc/cooker.git {% endhighlight %}

Step 2

npmでgruntのモジュール（node_modulesフォルダ）をワーキングディレクトリにインストール。

{% highlight text %} $ cd cooker $ npm install {% endhighlight %}

Step 3

goコマンドでgruntとbowerで関連ファイルのインストール→ビルド→サーバー起動。

{% highlight text %} $ grunt go {% endhighlight %}

Gruntfile.js

grunt command

Default

サーバー起動（+livereload）　→　ファイル監視（jekyll,less）

{% highlight text %} $ grunt {% endhighlight %}

Build

ファイルの最適化 → ビルド

{% highlight text %} $ grunt build {% endhighlight %}

Test

grunt build → エラーチェック

{% highlight text %} $ grunt test {% endhighlight %}

Go

vendorファイルをインストール → grunt test → grunt build → grunt default

{% highlight text %} $ grunt go {% endhighlight %}

Grunt Task

gruntの主なタスク

  • livereloadx ブラウザのライブリロード（chromreのアドオン必要）
  • connect ローカルサーバー
  • esteWatch ファイル監視
  • bower ベンダーライブラリのインストール（インストールするライブラリはbower.jsonで管理）　
  • jshint jsエラーチェック
  • concat jsファイル結合
  • uglify jsファイルにクレジットバナー追加
  • validation hmtlエラーチェック
  • htmlmin html圧縮
  • csslint cssエラーチェック
  • usebanner cssファイルにクレジットバナー追加
  • less lessのコンパイル
  • csscomb cssの最適化
  • shell jekyllコマンド実行
  • copy ファイルコピー
  • notify growl通知
